Types

Empty type

A datatype without constructors is the empty type. This represents the falsehood.

data
  𝟘 (ℓ : Level) : Type ℓ
  where
  -- Nothing

Synonyms of ⊥:

⊥     = 𝟘
Empty = 𝟘

Its eliminator principle also called Ex falso quodlibet:

exfalso
  : ∀ {ℓ₁ ℓ₂ : Level} {A : Type ℓ₂}
  → ⊥ ℓ₁
  --------
  → A

exfalso ()

Synonyms of exfalso rule:

Empty-elim = exfalso
⊥-elim     = exfalso
𝟘-elim     = exfalso

The negation symbol is a shortcut of an implication for the falsehood.

¬ : ∀ {ℓ : Level} → Type ℓ → Type ℓ
¬ {ℓ} A = (A → ⊥ ℓ)

Unit type

The unit type is defined as a record to get also the $η$-rule definitionally. This type has no elimination rule.

record
  𝟙 (ℓ : Level)
    : Type ℓ
  where
  constructor unit

-- {-# BUILTIN UNIT ⊤ #-}

Synonyms for the Unit type:

Unit = 𝟙
⊤    = 𝟙

Synonyms for the data constructor:

pattern ★ = unit
pattern ∗ = unit

∑-types

The dependent product is called Sigma types. It’s the the type for pairs where the second term in the pair may depend on the first term.

record
  ∑ {ℓ₁ ℓ₂ : Level} (A : Type ℓ₁)(B : A → Type ℓ₂)
   : Type (ℓ₁ ⊔ ℓ₂)
  where
  constructor _,_
  field
    π₁ : A
    π₂ : B π₁

infixr 60 _,_
open ∑ public

Symbol synonym:

Σ = ∑ -- \Sigma and \sum

Constructor synonyms:

proj₁ = π₁
proj₂ = π₂

pr₁   = π₁
pr₂   = π₂

fst   = π₁
snd   = π₂

#     =  π₁

Π-types

Π
  : ∀ {ℓ₁ ℓ₂ : Level} (A : Type ℓ₁)(B : A → Type ℓ₂)
  --------------------------------------------------
  → Type (ℓ₁ ⊔ ℓ₂)

Π A B = (x : A) → B x

Synonyms

∏ = Π   -- \prod vs \Pi

Products

Product type as a particular case of the Sigma type.

_×_
  : ∀ {ℓ₁ ℓ₂ : Level} (A : Type ℓ₁)(B : Type ℓ₂)
  ----------------------------------------------
  → Type (ℓ₁ ⊔ ℓ₂)

A × B = ∑ A (λ _ → B)

infixl  39 _×_

Coproducts

Sum types as inductive types

data
  _+_ {ℓ₁ ℓ₂ : Level} (A : Type ℓ₁)(B : Type ℓ₂)
    : Type (ℓ₁ ⊔ ℓ₂)
  where
  inl : A → A + B
  inr : B → A + B

infixr 31 _+_

Constructors synonyms:

-- pattern left  = inl  -- TODO: This is not working as I expected
-- pattern right = inr

The elimination principle:

+-elim
  : ∀ {ℓ₁ ℓ₂ ℓ₃ : Level} {A : Type ℓ₁}{B : Type ℓ₂} {C : Type ℓ₃}
  → (A → C) → (B → C)
  -------------------
  → (A + B) → C

+-elim A→C B→C (inl x) = A→C x
+-elim A→C B→C (inr x) = B→C x

Synonyms:

cases = +-elim

syntax cases f g = ⟨ f + g ⟩

Implications

data
  _⇒_ {ℓ₁ ℓ₂ : Level} (A : Type ℓ₁)(B : Type ℓ₂)
    : Type (ℓ₁ ⊔ ℓ₂)
  where
  fun : (A → B) → A ⇒ B

Bi-implications

_⇔_
  : ∀ {ℓ₁ ℓ₂}
  → Type ℓ₁ → Type ℓ₂
  -------------------
  → Type (ℓ₁ ⊔ ℓ₂)

A ⇔ B = (A → B) × (B → A)

Synonyms:

_↔_ = _⇔_

    𝟙₂ : 𝟚 ℓ

Synonyms:

Bool = 𝟚 lzero

Constructors synonyms:

false : 𝟚 lzero
false = 𝟘₂

true : 𝟚 lzero

Natural numbers

data
  ℕ : Type lzero
  where
  zero : ℕ
  succ : ℕ → ℕ

Synonyms for natural numbers

Nat = ℕ

pattern zr = zero
pattern sc = succ

{-# BUILTIN NATURAL ℕ #-}

An order relation will use in the following type constructor.

module ℕ-< {ℓ : Level} where
  _<_ :  ℕ → ℕ → Type ℓ
  zero   < zero   = ⊥ ℓ
  zero   < succ b = ⊤ ℓ
  succ _ < zero   = ⊥ ℓ
  succ a < succ b = a < b

and we can state the relation $\geq$ as as shortcut for…

  _>_ : ℕ → ℕ → Type ℓ
  a > b = b < a

Finite sets

A finite set can be define in so many ways. We opt to use a ∑-type which we believe is clear enough to say what they are. A finite set of $n : \mathsf{N}$, $\mathsf{Fin}_{n}$, is the collection of numbers less ($<$) than the number $n$. This notion is the following type family.

module Fin (ℓ : Level) where
  mutual
    Fin : ℕ → Type ℓ
    Fin n = Σ (ℕ) (λ m → m < n)
    open ℕ-< {ℓ}

Even though, there are other approaches to define finite sets, (in the standard-library in Agda, they have defined inductively fin sets as with natural numbers.)

module _ {ℓ : Level} where
  ⟦_⟧ : ℕ → Type ℓ
  ⟦_⟧ zero  = 𝟘 ℓ
  ⟦_⟧ (succ n)  = 𝟙 ℓ + ⟦ n ⟧

In math books, we denote by the finite set of $n$ as $[n]$, we mimic this notation as follows:

Synomym:

  Fin₂ = ⟦_⟧

Without going further, it’s natural to define two essential functions: successor, and predecessor.

Succesor function on (finite) natural numbers are well-defined when we consider sets with at least one element.

  • $1 :≡ inl unit$

  • $n :≡ inr (inr …)$

  ⟦⟧-succ
    : {n : ℕ}
    → ⟦ n ⟧ → ⟦ succ n ⟧

  ⟦⟧-succ {succ n} (inl x) = inr (inl unit)
  ⟦⟧-succ {succ n} (inr x) = inr (⟦⟧-succ x)

  ⟦⟧-pred
    : ∀ (n : ℕ)
    → ⟦ n ⟧ →  ⟦ n ⟧ -- clarify why in succ?,

  ⟦⟧-pred (succ n) (inl x) = inl x
  ⟦⟧-pred (succ n) (inr x) = inr (⟦⟧-pred n x)

Equalities

In HoTT, we have a different interpretation of type theory in which the set-theoretical notion of sets for types is replaced by the topological notion of spaces.

The (homogeneous) equality type also called identity type is considered a primary type (included in the theory by default). To form this type, we fix a type A and a term a : A, to have the identity type a == a, also denoted by Id(a,a) or a⇝a. We only have one constructor of these types called idp or sometimes refl. To use identity types, we have below the J-eliminator.

data
  _==_ {ℓ : Level}{A : Type ℓ} (a : A)
    : A → Type ℓ
  where
  idp : a == a

-- synonyms for the identity type
Eq   = _==_
Id   = _==_
Path = _==_
_⇝_  = _==_   -- '\r~'
_≡_  = _==_   -- '\equiv' (may be not good idea...)

infix 30 _==_ _⇝_ _≡_

{-# BUILTIN EQUALITY _==_  #-}

refl
  : ∀ {ℓ : Level} {A : Type ℓ}
  → (a : A)
  -----------------
  → a == a

refl  a = idp

Symmetry property for the identity types.

sym
  : ∀ {ℓ : Level} {A : Type ℓ} {x y : A}
  → x == y
  --------
  → y == x

sym idp = idp

syntax sym p = − p

To work with identity types, we have its induction principle as the J-eliminator.

Paulin-Mohring J rule

J
  : ∀ {ℓ : Level} {A : Type ℓ}  {a : A} {ℓ₂ : Level}
  → (B : (a' : A) (p : a == a') → Type ℓ₂)
  → (d : B a idp)
  ----------------------------------------
  → {a' : A} (p : a == a') → B a' p

J {a = a} B d idp = d

J'
  : ∀ {ℓ : Level} {A : Type ℓ}  {a : A} {ℓ₂ : Level}
  → (B : (a' : A) (p : a' == a) → Type ℓ₂)
  → (d : B a idp)
  ----------------------------------------
  → {a' : A} (p : a' == a) → B a' p

J' {a = a} B d idp = d

Decidable type

data
  Dec {ℓ : Level}(P : Type ℓ)
    : Type ℓ
  where
  yes : ( p : P) → Dec P
  no  : (¬p : P → ⊥ ℓ) → Dec P

⌊_⌋ : ∀ {ℓ : Level}{P : Type ℓ} → Dec P → 𝟚 ℓ
⌊ yes _ ⌋ = 𝟙₂
⌊ no  _ ⌋ = 𝟘₂

REL
  : ∀ {ℓ : Level}
  → Type ℓ → Type ℓ
  ----------------
  → Type (lsuc ℓ)

REL {ℓ} A B = A → B → Type ℓ

Decidable
  : ∀ {ℓ : Level} {A B : Type ℓ}
  → REL A B
  → Type ℓ

Decidable _∼_ = ∀ x y → Dec (x ∼ y)
